- **Purpose:** This is your "Zettelkasten" or "Evergreen Notes" area for atomic facts, definitions, algorithms, data structures, design patterns, theoretical principles, etc. This is where the bulk of your learning goes and where you connect ideas.
    
- **Structure:**
    
    - **Flat:** A single folder with all your concept notes. This works well because you'll rely on linking and MOCs (Maps of Content) rather than strict folder hierarchy.
        
    - **Lightly Categorized (Optional):** If you prefer, you can have broad subfolders, but try to keep them high-level. Avoid deep nesting.
        
        - `30_Concepts/Algorithms/`
            
        - `30_Concepts/Data_Structures/`
            
        - `30_Concepts/Operating_Systems/`
            
        - `30_Concepts/Networking/`
            
        - `30_Concepts/Programming_Languages/Python/`
            
- **Content (Atomic Notes are key here):**
    
    - **Definition notes:** "What is a Binary Search Tree?"
        
    - **Concept notes:** "Recursion vs Iteration," "Big O Notation."
        
    - **Algorithm notes:** "Dijkstra's Algorithm" (with pseudo-code, complexity, use cases, related algorithms).
        
    - **Pattern notes:** "Observer Pattern," "Singleton Pattern."
        
    - **Problem-solving techniques:** "Divide and Conquer."
        
    - **Each note should focus on ONE core idea.**
        
    - **Heavily interlinked:** Link to prerequisites, related concepts, examples, and applications.
        
    - **Your own words:** Rewrite information from sources in your own understanding.
        
    - **Code Snippets:** Embed small, illustrative code snippets directly (use code blocks).
        
    - **Diagrams/Visuals:** Link to or embed relevant diagrams (e.g., from Excalidraw, Mermaid.js, or external images).
