# Service Oriented Architecture (SOA)

**SOA (Service-Oriented Architecture)** is a software design approach where applications are built by combining **loosely coupled, reusable services**. Each **service** is a self-contained unit of functionality (like user authentication, payment processing, or data retrieval) that can be **accessed over a network**, typically via standard protocols like HTTP or SOAP.

### 🔑 Key characteristics:

- **Modularity**: Applications are broken into independent services.
    
- **Interoperability**: Services communicate over standard interfaces (e.g., APIs), even if written in different languages.
    
- **Reusability**: Services can be reused in different applications or contexts.
    
- **Scalability & Maintainability**: Each service can be scaled or updated independently.
    

### 🧱 Example:

A shopping app using SOA might have:

- A **User Service** (for authentication),
    
- A **Product Service** (to list items),
    
- A **Payment Service** (to handle transactions).
    

These services work together but can evolve separately.
