This approach involves having a specific folder (e.g., `00_Inbox` or `Inbox`) where new, unorganised notes are initially created. You might have a single "Inbox.md" note within that folder, or you might create a new, distinct note for each incoming idea.

**How it works:**

- You set your default new note location to this "Inbox" folder in Obsidian settings.
    
- When a new idea comes, you create a _new note_ (e.g., `Ctrl/Cmd + N`), give it a quick, descriptive title, and dump the idea there.
    
- Alternatively, you can have a single `Inbox.md` note where you append new thoughts to a bulleted list.
    

**Pros of using a dedicated Inbox folder/note:**

- **Clear Separation:** It explicitly separates "raw capture" from your more organized notes.
    
- **Easier to Process (for some):** Having separate files for each idea might make processing more manageable, as you can easily drag and drop files to their final destinations or delete them.
    
- **Less Daily Note Clutter:** Keeps your Daily Notes cleaner if you prefer them for journaling or pure daily logging.
    

**Cons of using a dedicated Inbox folder/note:**

- **Slightly More Friction:** You have to think of a file name for each new note, even if it's just a quick "Idea 2025-07-30". This adds a tiny bit of friction compared to just typing into an open Daily Note.
    
- **Loss of Chronological Context:** Individual inbox notes don't inherently have the same chronological context as a Daily Note. You'd need to manually add dates or timestamps if that's important.
    
- **Potential for "Note Bloat":** If you create a new note for every fleeting thought, your inbox folder can quickly become huge with many small, unprocessed notes.
